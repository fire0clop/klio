# Frontend Roadmap — Swift / iOS

## Стек
- **Swift 5.10+** + **SwiftUI**
- **Xcode 16+**
- **Swift Concurrency** (async/await) — сетевые запросы
- **SwiftData** — локальный кеш
- **Swift Charts** — графики аналитики
- Минимальная версия iOS: **17.0**

---

## Фаза 1 — Архитектура и основа (неделя 1–2)

### 1.1 Проект и структура
- [ ] Создать Xcode проект (SwiftUI App)
- [ ] Архитектура: **MVVM**
- [ ] Структура папок:
  ```
  App/
  ├── Core/
  │   ├── Network/        # APIClient, endpoints, interceptors
  │   ├── Storage/        # SwiftData stack, Keychain wrapper
  │   └── Router/         # NavigationStack маршруты
  ├── Features/
  │   ├── Auth/
  │   ├── Onboarding/     # профиль + первая цель
  │   ├── GoalCreation/   # AI-диалог при создании цели
  │   ├── Dashboard/
  │   ├── CheckIn/
  │   ├── Analytics/
  │   └── Profile/
  └── Components/         # переиспользуемые UI-компоненты
  ```
- [ ] `APIClient` — async/await, JWT refresh, обработка ошибок
- [ ] Keychain wrapper для хранения токенов
- [ ] `AppRouter` с enum-маршрутами, разделение Auth flow / Main flow

### 1.2 Tab структура (Main flow)
- [ ] TabView с 4 табами: **Сегодня / Цели / Аналитика / Профиль**
- [ ] "Сегодня" — главный экран, открывается по умолчанию

## Фаза 2 — Аутентификация и онбординг (неделя 3–4)

### 2.1 Auth экраны
- [ ] Экран регистрации (email + пароль)
- [ ] Экран входа
- [ ] Автологин при запуске если токен валиден

### 2.2 Онбординг — профиль
Показывается один раз при первом входе, 2 шага:
- [ ] Шаг 1: имя, возраст, пол
- [ ] Шаг 2: рост, вес
- [ ] Progress-индикатор, кнопка "назад"
- [ ] После заполнения — сразу переход к созданию первой цели
- [ ] Никакой длинной анкеты — только эти 4 параметра

## Фаза 3 — Создание цели с AI-диалогом (неделя 5–6)

Это ключевой и самый важный экран приложения.

### 3.1 Экран создания цели
- [ ] Текстовое поле: "Какую цель хочешь поставить?" — свободный ввод
- [ ] После отправки — появляется индикатор "AI анализирует..."
- [ ] Далее — чат-интерфейс: вопросы AI появляются как пузыри сообщений
- [ ] Ответы пользователя — текстовый ввод или кнопки-варианты (AI решает что уместнее)
- [ ] 2–4 вопроса максимум, потом "Всё понял, создаю цель..."
- [ ] Анимированное появление карточки созданной цели с первыми эффектами

### 3.2 Расписание цели
Выбирается до начала диалога или после, отдельным шагом:
- [ ] **Каждый день** — дефолт
- [ ] **Каждые N дней** — степер
- [ ] **Дни недели** — мультиселект пн/вт/ср/чт/пт/сб/вс
- [ ] **X раз в неделю** — степер без привязки к дням

### 3.3 UX-детали
- [ ] Если пользователь не хочет отвечать на вопрос — кнопка "Пропустить"
- [ ] История диалога видна на экране (scroll вверх)
- [ ] После создания — конфетти / анимация успеха

## Фаза 4 — Dashboard "Сегодня" (неделя 7)

Главный экран — открывается каждый день.

### 4.1 Структура экрана
- [ ] Приветствие с именем и датой
- [ ] Секция **Цели на сегодня** — только те цели у которых сегодня плановый день
  - Карточка цели: название, streak, чекбокс выполнено/нет
  - Если у цели нет плановых целей сегодня — она не показывается
- [ ] Секция **Мои показатели** — универсальный дневной лог
  - Поля: вес, сон, энергия (слайдер 1–10), настроение (слайдер 1–10)
  - В первые 3 дня секция скрыта, потом появляется с hint-баннером
- [ ] Кнопка **Сохранить день** — фиксирует всё разом

### 4.2 Состояния экрана
- [ ] Пустое состояние: нет целей на сегодня → CTA "добавить цель"
- [ ] Все отмечено: анимация успеха, показ текущих эффектов дня
- [ ] День уже сохранён: режим просмотра с кнопкой "редактировать"

### 4.3 Специфические метрики
- [ ] Под каждой целью — опциональные поля метрик, предложенные AI
  - Пример: под "бегать" → поле "пульс после тренировки"
- [ ] Поля появляются только если пользователь хоть раз их заполнял или сам открыл

### 5.1 Экран "Цели"

- [ ] Карточка: название, расписание, текущий streak, % выполнения за 30 дней
- [ ] FAB кнопка "+" — добавить новую цель

### 5.2 Детали цели
- [ ] Страница цели при тапе:
  - Название, расписание, дата старта
  - Streak / best streak / всего выполнено дней
  - Calendar heatmap — тепловая карта по месяцам (зелёный/красный/серый)
  - Карточки эффектов с прогресс-барами
- [ ] Кнопки: редактировать расписание, архивировать цель

## Фаза 6 — Аналитика (неделя 9–10)

### 6.1 Экран "Аналитика"
- [ ] **Инсайты** (топ секция) — текстовые карточки от AI
  - Пример: "В дни когда ты выполнял все цели, твоя энергия была на 38% выше"
  - Обновляются раз в неделю
- [ ] **Эффекты по целям** — аккордеон по каждой цели
  - Иконка, название эффекта, прогресс-бар с процентом
  - Проценты растут со streak, немного откатываются при пропуске
- [ ] **Графики** (Swift Charts):
  - Линейный: энергия / настроение / вес по времени
  - Переключатель: 7 дней / 30 дней / всё время

### 6.2 UX-детали аналитики
- [ ] Скелетон при загрузке данных
- [ ] Пустое состояние если данных < 3 дней: "Приходи через несколько дней — появится аналитика"
- [ ] Milestone-карточки: "7 дней без сигарет — поздравляем!"

## Принципы UX приложения

- **Постепенный разогрев**: первые 3 дня — только галочки по целям. Метрики веса/сна/энергии появляются с hint-баннером на 4й день
- **AI как собеседник**: при создании цели — чат, не форма
- **Честные данные**: проценты эффектов мотивационные, но основаны на реальном streak пользователя
- **Плановые дни**: цель показывается только в день по расписанию, пропуск вне расписания не ломает streak
